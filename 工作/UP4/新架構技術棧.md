## 技術亮點  
  
### 架構層面  
  
- **DDD + Clean Architecture**: UP4 新系統採用領域驅動設計  
- **漸進式遷移**: 透過 Strangler Pattern 逐步遷移舊系統  
- **路徑分流**: `/api/**` (舊系統) vs `/up/api/**` (新系統)  
  
### 安全層面  
  
- **JWT Token**: 新系統用 JWT 認證  
- **Spring Security**: 統一安全配置  
- **@Login AOP**: 舊系統用 AOP 攔截式認證  
  
### 資料層面  
  
- **JPA + Hibernate**: 新系統 ORM  
- **SQL Server**: 資料庫  
- **RMI**: 舊系統呼叫 PASE  
  
### 工具層面  
  
- **Lombok**: 減少樣板代碼  
- **MapStruct**: DTO ↔ Entity 映射  
- **Hutool**: 工具函數庫  
- **SpringDoc**: Swagger 文檔自動生成

## 性能優化考量  
  
| 項目 | 現況 | 建議 |  
|-----|------|-----|  
| 快取 | Redis Starter 配置但可能未啟用 | 啟用 Redis 緩存層 |  
| 資料庫連線 | JDBC + JPA | 考慮連線池配置 |  
| RMI 開銷 | N+1 查詢問題 | 批量查詢或遷移到 UP4 |  
| JSON 性能 | 多個 JSON 庫並存 | 統一使用 Jackson |
