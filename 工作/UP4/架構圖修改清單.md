# 架構圖修改清單（簡潔版）  
  
## 原圖問題  
您的架構圖在 "Spring Boot 內路由" 部分寫著：  
```  
DispatcherServlet / 舊系統路由  
DispatcherServlet / 新系統路由  ← 暗示有兩個 servlet
```  
  
這會讓人誤解為有多個 `DispatcherServlet` 配置。  
  
---  
  
## ✅ 修改方向  
  
### 第 1 步：合併 DispatcherServlet  
**改之前**：  
```  
┌─ DispatcherServlet / 舊系統路由  
└─ DispatcherServlet / 新系統路由  
```  
  
**改之後**：  
```  
┌─ 單一 DispatcherServlet│  (Spring Boot 標準配置，無需自訂)  
```  
  
---  
  
### 第 2 步：新增 HandlerMapping 說明  
在 DispatcherServlet 下面加一層：  
  
```  
┌─ 單一 DispatcherServlet   ↓┌─ HandlerMapping (URL 路徑分流)  
│  ├─ /api/**      → Legacy Controllers│  └─ /up/api/**   → UP4 Controllers
```  
  
---  
  
### 第 3 步：修改「微服務日志」標籤  
**改之前**：  
```  
微服務日志 (Micrometers / Actuators)
```  
  
**改之後**：  
```  
安全與攔截層 (Security + Interceptor)├─ /api/**    → AuthorizationInterceptor + @Login AOP└─ /up/api/** → JWT Filter + @PreAuthorize
```  
  
---  
  
### 第 4 步：選擇性：用顏色區分  
- **藍色箭頭** = UP4 新系統 (`/up/api/**`)  
- **紅色箭頭** = Legacy 舊系統 (`/api/**`)  
  
---  
  
## 修改後的架構核心邏輯  
  
```  
Client Request  
    ↓  Tomcat    ↓【單一 DispatcherServlet】← 改這裡（合併成一個）  
    ↓【HandlerMapping】← 加這層（標出分流點）  
 /api/** ──→ Legacy  
 /up/api/** ──→ UP4  
    ↓ ↓【Security + Interceptor - 路徑規則】← 改這層（更新標籤和說明）  
    ↓ ↓【Controllers】  
    ↓ ↓【Services】  
    ↓ ↓【Persistence】  
    ↓ ↓【Database】  
```

## 驗證修改是否正確  
  
修改完後，檢查這幾點：  
  
- [ ] 只有一個 DispatcherServlet box（不是兩個）  
- [ ] 有明確的 "HandlerMapping" 層說明 URL 分流  
- [ ] `/api/**` 和 `/up/api/**` 的標籤清楚  
- [ ] 舊系統和新系統的流向用箭頭區分  
- [ ] 沒有暗示有多個 servlet 配置

|-----|-----|-------|

## 修改完成後  
  
建議在圖下方加上這段註解：  
  
> **架構說明**  > - 採用單一 Spring Boot 應用、單一 DispatcherServlet    
> - 透過 URL 路徑映射 (`HandlerMapping`) 區分新舊 API    
> - Security 與 Interceptor 在同一層，依路徑規則分流  > - `/api/**`：舊系統（WFCIService），由 @Login AOP 驗證  > - `/up/api/**`：新系統（UP Module），由 Spring Security JWT 驗證  > - 此設計便於維護，日後可升級為 API Gateway + 多應用架構
