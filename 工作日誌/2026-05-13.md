這段程式主要是在做：

1. 把 `actReg` 物件轉成 JSON 字串
    
2. 再把 JSON 字串轉成 `Map<String, Object>`
    
3. 最後補上一些額外欄位
    

等於是：

```text
Java Object
   ↓
JSON String
   ↓
Map<String, Object>
```

我拆成幾段說明。

---

## 1. 建立 Jackson 的 ObjectMapper

```java
ObjectMapper objectMapper = new ObjectMapper();
```

這是 Jackson Databind 的核心類別。

用途：

- Java Object ↔ JSON
    
- 序列化（Serialize）
    
- 反序列化（Deserialize）
    

---

## 2. 忽略 null 欄位

```java
objectMapper.setSerializationInclusion(Include.NON_NULL);
```

意思：

```java
欄位為 null 時，不要輸出到 JSON
```

例如：

```java
class User {
    String name = "John";
    String email = null;
}
```

正常 JSON：

```json
{
  "name": "John",
  "email": null
}
```

設定 `NON_NULL` 後：

```json
{
  "name": "John"
}
```

---

## 3. 註解的意思

```java
//ignore null value, JSONUtil.parseObj cannot convert null
```

代表作者以前可能使用：

```java
JSONUtil.parseObj(...)
```

（來自 Hutool）

但 Hutool 在某些情況下：

```java
null value
```

會造成轉換問題。

所以先：

```java
忽略 null
```

避免 parse 時炸掉。

---

## 4. Java Object → JSON String

```java
String jsonStr = objectMapper.writeValueAsString(actReg);
```

這行會把：

```java
actReg
```

轉成 JSON。

例如：

```java
actReg:
{
   id=1001,
   name="FlowRing"
}
```

變成：

```json
{
  "id":1001,
  "name":"FlowRing"
}
```

存在：

```java
jsonStr
```

---

## 5. JSON String → Map

```java
resMap = objectMapper.readValue(
    jsonStr,
    new TypeReference<Map<String, Object>>() {}
);
```

這行是重點。

它把：

```json
{
  "id":1001,
  "name":"FlowRing"
}
```

轉成：

```java
Map<String, Object>
```

結果：

```java
{
   "id" -> 1001,
   "name" -> "FlowRing"
}
```

---

## 為什麼不用直接轉？

很多人會問：

```java
為什麼不直接轉 Map？
```

例如：

```java
objectMapper.convertValue(...)
```

其實可以。

你這段比較像：

```text
Object
 → JSON
 → Map
```

是「中轉 JSON」的方式。

---

## 更好的寫法（推薦）

其實這段可以簡化成：

```java
resMap = objectMapper.convertValue(
    actReg,
    new TypeReference<Map<String, Object>>() {}
);
```

這樣：

- 少一次 JSON 字串建立
    
- 效能更好
    
- 不需要 serialize + deserialize
    

---

## 6. 加入額外欄位

```java
resMap.put("name", actName);
resMap.put("isRegister", "false");
```

等於在 Map 裡補資料：

```java
{
   "id":1001,
   "name":"xxx",
   "isRegister":"false"
}
```

注意：

```java
"isRegister", "false"
```

這是字串 `"false"`。

不是：

```java
false
```

布林值。

這兩個不同。

目前你的 Map 會變成：

```json
{
  "isRegister":"false"
}
```

而不是：

```json
{
  "isRegister":false
}
```

如果你想要真正 boolean：

```java
resMap.put("isRegister", false);
```

---

## 7. Exception 處理

```java
catch (JSONException e)
```

這是：

- Hutool
    
- org.json
    
- fastjson
    

那類 JSON Exception。

但你這段程式其實主要是 Jackson：

```java
ObjectMapper
```

Jackson 通常丟的是：

```java
JsonProcessingException
```

所以：

```java
catch (JSONException e)
```

很可能已經不需要了。

---

## 這段程式整體目的

本質上是在做：

```text
把 actReg 動態轉成 Map
方便後面自由塞欄位
```

因為：

```java
Map<String, Object>
```

比 Java class 更自由。

常見用途：

- API response
    
- 動態 JSON
    
- 前端回傳格式
    
- 不固定欄位
    

---

## 建議改寫（較乾淨）

```java
try {
    ObjectMapper objectMapper = new ObjectMapper();

    objectMapper.setSerializationInclusion(Include.NON_NULL);

    Map<String, Object> resMap =
        objectMapper.convertValue(
            actReg,
            new TypeReference<Map<String, Object>>() {}
        );

    resMap.put("name", actName);
    resMap.put("isRegister", false);

} catch (JsonProcessingException e) {
    logger.error(e.getMessage(), e);
}
```

甚至：

```java
convertValue
```

通常連 `JsonProcessingException` 都不一定需要 catch。






```
以下兩種寫法有什麼差別嗎? 哪一個較佳?  
String currencyTypeName = (String) storeObject.get("currencyType");
String currencyTypeName = storeObject.get("currencyType").toString();

```
這兩種寫法差很多，尤其是在：

- `null` 處理
    
- 型別安全
    
- 實際資料型態
    

都有差異。

---

# 第一種

```java
String currencyTypeName =
    (String) storeObject.get("currencyType");
```

這是：

```text
強制型別轉換（Cast）
```

意思：

```text
我很確定這個物件本來就是 String
```

---

## 例如

```java
Map<String, Object> map = new HashMap<>();
map.put("currencyType", "USD");
```

這樣：

```java
(String) map.get("currencyType")
```

沒問題。

---

# 第二種

```java
String currencyTypeName =
    storeObject.get("currencyType").toString();
```

這是：

```text
不管原本型別是什麼
都轉成字串
```

---

## 例如

```java
map.put("currencyType", 100);
```

結果：

```java
"100"
```

因為：

```java
Integer.toString()
```

會被呼叫。

---

# 最大差異：null

這才是真正重點。

---

## 第一種遇到 null

```java
map.put("currencyType", null);
```

```java
String value = (String) map.get("currencyType");
```

結果：

```java
value == null
```

安全。

不會 exception。

---

## 第二種遇到 null

```java
String value =
    map.get("currencyType").toString();
```

等於：

```java
null.toString();
```

直接：

```text
NullPointerException
```

---

# 再來是型別安全

---

## 第一種

```java
(String)
```

要求：

```text
原本就必須是 String
```

否則：

```text
ClassCastException
```

例如：

```java
map.put("currencyType", 123);
```

```java
(String) 123
```

炸：

```text
ClassCastException
```

---

## 第二種

```java
123.toString()
```

合法。

結果：

```java
"123"
```

---

# 比較表

|情況|`(String)`|`.toString()`|
|---|---|---|
|值是 String|OK|OK|
|值是 Integer|ClassCastException|"123"|
|值是 null|回傳 null|NullPointerException|
|型別要求|嚴格|寬鬆|
|安全性|型別安全|容易 NPE|

---

# 哪個較佳？

其實看你的需求。

---

# 情境 1：你「確定」它就是 String

推薦：

```java
(String)
```

因為：

- 型別明確
    
- 比較符合語意
    
- 不會偷偷把 Integer 轉字串
    

例如：

```java
Map<String, Object>
```

其實設計上：

```text
currencyType 就應該是 String
```

那用 cast 比較合理。

---

# 情境 2：你只是想拿字串顯示

例如：

```text
log
畫面顯示
debug
```

可以：

```java
Objects.toString(obj, "")
```

或：

```java
String.valueOf(obj)
```

---

# 更推薦的安全寫法

## 最安全：

```java
String currencyTypeName =
    Objects.toString(
        storeObject.get("currencyType"),
        null
    );
```

或：

```java
String currencyTypeName =
    String.valueOf(
        storeObject.get("currencyType")
    );
```

null
